# SolverWatch Insights — public data layer

This repo is the public data backend for [solverwatch.com](https://solverwatch.com).

The marketing site fetches `manifest.json` from this repo on every page load (ISR, 1-hour
revalidation), and individual article markdown files on demand. No auth, no rate limiting
beyond GitHub's CDN.

## Layout

```
manifest.json                       # index of all articles, newest first
articles/
  <date>-<slug>.md                  # one file per article, front-matter + body
.github/workflows/daily.yml         # cron: generates 10 articles per day
tools/
  generate-daily.py                 # the generator (template-driven, no LLM)
```

## How the cron works

Every day at 04:00 UTC, GitHub Actions runs `tools/generate-daily.py`:

1. Hits the public **iTunes Search API** (`itunes.apple.com/search`) with rotating queries.
2. Picks 10 article templates (listicle, hidden gems, trending, spotlight, comparison, etc.)
3. Generates unique markdown with real app metadata — no LLM cost, always unique.
4. Writes new `articles/<slug>.md` files and prepends entries to `manifest.json`.
5. Commits + pushes. Public, so solverwatch.com picks it up on next ISR revalidation.

## Adding a new article template

1. Add a new template constant in `tools/generate-daily.py` (e.g. `BUZZ_TEMPLATES = [...]`)
2. Write a new generator function (e.g. `article_buzz(...)`)
3. Add it to the `GENERATORS` list near the bottom of the file
4. Optionally add a new label in the marketing site's `lib/insights.ts`

## Article attribution

Every article ends with:
> *Generated by the **minimax cron job** at YYYY-MM-DD 04:00 UTC.*

That's intentional — we want visitors (and crawlers) to see the source.

## Cost

- iTunes Search API: free
- GitHub Actions: ~3 minutes of free-tier per day
- LLM: none — pure data transformation
- Hosting: GitHub's free public-repo CDN

Total recurring cost: **$0**.
